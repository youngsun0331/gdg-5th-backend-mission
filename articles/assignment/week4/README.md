## 1번째 미션 : 예외처리, AOP, 리팩토링 원칙 학습하기

### 개요
예외처리, AOP, 리팩토링 원칙에 대해 학습하며 유지보수하기 좋은 코드를 작성할 수 있는 기반을 다지는 미션입니다.
2번째 미션에서 실제 코드에 적용할 수 있도록 개념과 예시를 이해하는 것을 목표로 합니다.

### 미션 내용
리팩토링 과정 자체는 코드 바이 코드, 사람 바이 사람이기 때문에
여러분들이 관심이나 흥미가 가는 내용에 대해서 자유롭게 학습하고 articles/assignment/week4/refactoring.md에 작성해주시면 됩니다.

아래는 제가 추천하는 몇가지 예시들입니다!

- 예외처리

체크 / 언체크 예외, 비즈니스 / 시스템 예외 구분
전역 예외 처리 방식(@ControllerAdvice 등)과 에러 응답 포맷 설계 개념

- AOP

핵심 용어(Pointcut, Advice, Aspect 등)와 Spring AOP 동작 방식
로깅, 성능 측정, 트랜잭션 등 공통 관심사 분리 아이디어

- 리팩토링 원칙

Clean Code, SOLID, 중복 제거(DRY), 단순함(KISS), YAGNI 개념
“나쁜 코드 vs 개선 코드” 예시에 대해서 알아보기

## 2번째 미션 : 학습한 내용을 적용하여 1~3주차 코드 리팩토링하기

### 개요
1번째 미션에서 공부한 예외처리, AOP, 리팩토링 원칙을 실제 1~3주차 코드에 적용하여 구조를 개선하는 미션입니다.
기능은 그대로 유지하면서 코드 품질, 일관성, 유지보수성을 높이는 것이 목표입니다.

### 미션 내용
1번째 미션에서 공부한 내용을 바탕으로 1~3주차 코드를 자유롭게 리팩토링하여 올려주세요.

아래는 제가 추천하는 몇가지 예시들입니다!

- 예외처리 구조 개선

공통 에러 응답 포맷 정의 후 모든 API 에 적용
도메인별 Custom Exception 도입 및 전역 예외 처리 핸들러 적용

- AOP 적용

로깅, 실행 시간 측정 등 공통 로직을 AOP 로 분리
중복된 공통 처리 코드를 제거하고 각 클래스의 책임을 명확히 함

- 설계 및 코드 구조 리팩토링

단일 책임 원칙 위반 메서드/클래스 분리
중복 로직 공통 컴포넌트로 추출, 계층 간 의존 관계 정리
